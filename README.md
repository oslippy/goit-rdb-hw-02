# Крок 1 — 1NF

## Проблема

Поле **"Назва_товару і кількість"** містить множинні значення (порушення атомарності).

## Рішення

Розбиваємо кожен запис на окремі рядки.

## Результат

| Номер_замовлення | Назва_товару | Кількість | Адреса        | Дата       | Клієнт    |
| ---------------- | ------------ | --------- | ------------- | ---------- | --------- |
| 101              | Лептоп       | 3         | Хрещатик 1    | 2023-03-15 | Мельник   |
| 101              | Мишка        | 2         | Хрещатик 1    | 2023-03-15 | Мельник   |
| 102              | Принтер      | 1         | Басейна 2     | 2023-03-16 | Шевченко  |
| 103              | Мишка        | 4         | Комп’ютерна 3 | 2023-03-17 | Коваленко |

Усі атрибути атомарні → **1NF досягнута**

# Крок 2 — 2NF

## Проблема

Складений ключ:

```
(Номер_замовлення, Назва_товару)
```

Часткові залежності:

* `Номер_замовлення → Дата, Клієнт, Адреса`

## Рішення

Розділення таблиці на окремі сутності.

---

## Таблиця `order`

| id  | client_id | address_id | order_date |
| --- | --------- | ---------- | ---------- |
| 101 | 1         | 1          | 2023-03-15 |
| 102 | 2         | 2          | 2023-03-16 |
| 103 | 3         | 3          | 2023-03-17 |

---

## Таблиця `order_item`

| order_id | item_id | quantity |
| -------- | ------- | -------- |
| 101      | 1       | 3        |
| 101      | 2       | 2        |
| 102      | 3       | 1        |
| 103      | 2       | 4        |

---

## Таблиця `item`

| id | name    |
| -- | ------- |
| 1  | Лептоп  |
| 2  | Мишка   |
| 3  | Принтер |

---

Всі атрибути залежать від повного ключа → **2NF досягнута**

---

# Крок 3 — 3NF

## Проблема

Транзитивні залежності:

```
order → client → lastname
order → address → street, house
```

## Рішення

Виділення окремих таблиць.

---

## Таблиця `client`

| id | lastname  |
| -- | --------- |
| 1  | Мельник   |
| 2  | Шевченко  |
| 3  | Коваленко |

---

## Таблиця `address`

| id | street_id | house_id |
| -- | --------- | -------- |
| 1  | 1         | 1        |
| 2  | 2         | 2        |
| 3  | 3         | 3        |

---

## Таблиця `street`

| id | name        |
| -- | ----------- |
| 1  | Хрещатик    |
| 2  | Басейна     |
| 3  | Комп’ютерна |

---

## Таблиця `house`

| id | number |
| -- | ------ |
| 1  | 1      |
| 2  | 2      |
| 3  | 3      |

---

Відсутні транзитивні залежності → **3NF досягнута**

---

# Фінальна схема

| Таблиця    | Поля                                  |
| ---------- | ------------------------------------- |
| order      | id, client_id, address_id, order_date |
| order_item | order_id, item_id, quantity           |
| item       | id, name                              |
| client     | id, lastname                          |
| address    | id, street_id, house_id               |
| street     | id, name                              |
| house      | id, number                            |
